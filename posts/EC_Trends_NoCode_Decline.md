# 導入：SaaSの「壁」に絶望したことはあるか？

「Shopifyで月商1000万までは一瞬だった。でも、そこからが地獄だった」

最近、現場でよく聞く話です。
ShopifyやSalesforce Commerce Cloud (SFCC) などのSaaS/パッケージ製品は、ECサイト立ち上げのハードルを劇的に下げました。ノーコードでストアが建ち、アプリを入れれば機能が増える。エンジニアなんていらない、とすら言われました。

しかし、2025年。私たちは「SaaSの壁」に直面しています。
「アプリを20個入れたらサイトが激重になった（Plugin Hell）」
「SFCCのちょっとした改修見積もりが、高級車一台分だった」
「やりたい販促施策が、プラットフォームの仕様で実現できない」

便利さと引き換えに手放した「自由」を、取り戻すべき時が来ました。
この記事では、ノーコード全盛期の次に訪れた潮流「Composable Commerce」と、そこでエンジニアが果たすべき役割について解説します。

![Composable Tech](/Engineerblog/globe.svg)

# EC技術トレンド：揺り戻しと進化

## 1. The Decline of "Monolithic" No-Code
誤解を恐れずに言えば、大規模ECにおいて「全部入りの巨大SaaS（モノリス）」は限界を迎えています。
ノーコードは素晴らしいですが、それは「80点のサイト」を最速で作るための道具です。競争が激化するEC市場で、残りの「差別化の20点」を作り込もうとした瞬間、ノーコードは足枷になります。
「アプリ同士のデータが連携できない」「決済フローがいじれない」。このイライラは、ツールの限界ではなく、アーキテクチャの限界です。

## 2. Composable Commerce & MACH
そこで台頭しているのが、**Composable Commerce（コンポーザブル・コマース）**です。
ECサイトを「カート」「決済」「CMS」「検索」「在庫管理」といった部品（コンポーネント）に分解し、APIでつなぎ合わせる（Compose）考え方です。

*   **M**icroservices: 機能単位で独立させる。
*   **A**PI-first: 全ての連携をAPIで行う。
*   **C**loud-native: 柔軟なスケーリング。
*   **H**eadless: フロントとバックの完全分離。

「検索機能だけは爆速のAlgoliaを使いたい」「CMSは使い慣れたContentfulがいい」。
この**「いいとこ取り」**が可能になるのが、Composableの最大の強みです。

## 3. Headlessの一般化と「フロントエンドの復権」
バックエンド（Shopifyなど）はそのまま使い、フロントエンド（お客様が見る画面）だけをNext.jsなどでスクラッチ開発する「Headless Commerce」が一般化しました。
これにより、Core Web Vitals（表示速度）を極限まで高めたり、ネイティブアプリのようなヌルヌル動くUXを実現したりできます。
「画面を作る」というエンジニアの本質的な仕事が、より高度な形で戻ってきました。

## 4. Agentic Commerce 〜AIが「買う」時代〜
2025年の隠れたトレンドは、AIエージェントです。
これまでの「レコメンド」は人間への提案でしたが、これからはSiriやChatGPTのような**AIエージェントがユーザーの代わりに商品を探し、比較し、決済まで行う**可能性があります。
この時重要なのは「人間が見やすいUI」ではなく「AIが読みやすいAPI」です。構造化データとAPIの整備は、SEO対策以上に重要な「AIO（AI Optimization）」になります。

# 警鐘：「何でもComposable」は火傷する

ここで冷静になりましょう。エンジニアとして警鐘を鳴らしておきます。
**「流行りだからComposable」で、年商数億円規模のサイトを作ってはいけません。**

Composableは、自由と引き換えに「複雑性」という高いコストを要求します。
APIの接続管理、認証フローの整備、インフラの監視…。これらを自社で賄える開発力と資金力がある企業（ユニクロやNikeクラス）にとってのみ、Composableは「銀の弾丸」になります。

年商100億以下の多くの企業にとって、**正解はまだ「Shopify Plus」**である可能性が高いです。
「できないこと」があるからといって、安易にフルスクラッチに戻るのは、カーゴ・カルト（形式だけの模倣）です。

![Developer Role](/Engineerblog/next.svg)

# 結論：エンジニアは「接着剤」になれ

2025年のECエンジニアの役割は変わります。
ゼロからカートシステムを作る職人は、もういりません。

求められるのは、**SaaSとSaaSを繋ぐ「接着剤（Glue Code）」となれるエンジニア**です。

*   Shopifyの弱点を、AWS Lambdaで補完する。
*   Next.jsで最高のフロントエンドを作り、バックエンドのShopifyと繋ぐ。
*   バラバラのSaaSからデータを集め、BigQueryで分析基盤を作る。

ノーコードか、フルスクラッチか。そんな二元論は終わりです。
ノーコードの巨人の肩に乗り、手の届かないラストワンマイルを、あなたのコードで埋める。
「作る」から「組む」へ。それが、これからのEC開発のニュースタンダードです。
